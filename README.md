# Local File Manager

> 一个用浏览器打开的本地文件管理器，专治「用不惯 Mac 访达」的各种不爽。

## 你是否也有这些困扰？

Mac 的访达（Finder）对很多从 Windows/Linux 转过来的用户来说，体验并不友好：

- **点击区域太小**：访达的文件项点击热区有限，经常点了没反应，需要精确点到文件名或图标上
- **缺少地址栏直接编辑**：无法像在 Windows 资源管理器里那样直接在路径栏输入路径跳转
- **操作效率低**：右键菜单层级深，常用操作（新建文件夹、重命名、删除）不够直观
- **文件预览受限**：代码文件、配置文件等文本文件预览能力有限
- **隐藏文件切换麻烦**：每次都要记 `Cmd + Shift + .` 快捷键
- **多窗口管理混乱**：开多了访达窗口，找文件变成窗口捉迷藏

如果你也有同感，这个项目就是为你准备的。

## 这是什么？

**Local File Manager** 是一个运行在本地 `localhost` 的文件管理器，用浏览器打开即可使用。

不安装任何客户端，不上传任何文件到云端——它只在你自己的电脑上运行，管理你自己的文件。

![demo](https://img.shields.io/badge/status-demo%20ready-brightgreen)

## 功能

| 功能 | 说明 |
|------|------|
| 目录浏览 | 双击进入，面包屑路径导航，前进/后退/上级目录 |
| 文件搜索 | 实时搜索当前目录及一层子目录 |
| 文件预览 | 文本/代码文件内容预览，图片直接展示 |
| 文件操作 | 新建文件夹、重命名、删除（右键菜单） |
| 隐藏文件 | 一键切换显示/隐藏，不再需要记快捷键 |
| 视图切换 | 列表视图 / 网格视图 |
| 键盘快捷键 | `Alt+←/→` 前后导航，`Alt+↑` 上级目录，`Backspace` 后退 |
| 多语言 | 内置中英双语翻译层，可独立开关 |

## 快速开始

### 环境要求

- Python 3.8+
- macOS / Linux / Windows

### 安装与运行

```bash
# 1. 克隆仓库
git clone https://github.com/zjverse/local-file-manager.git
cd local-file-manager

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动
python3 main.py
```

浏览器打开 **http://127.0.0.1:8765** 即可使用。

默认从你的 Home 目录开始浏览。

## 技术栈

- **后端**: Python + FastAPI
- **前端**: 原生 HTML/CSS/JS（零构建工具，零框架依赖）
- **翻译层**: 独立的 `i18n.js`，通过 MutationObserver 运行时注入，不侵入原始代码

## 项目结构

```
local-file-manager/
├── main.py            # 启动入口
├── server.py          # FastAPI 后端
├── requirements.txt   # 依赖
├── static/
│   ├── index.html     # 页面
│   ├── style.css      # 样式
│   ├── app.js         # 前端逻辑
│   └── i18n.js        # 独立翻译层
└── CLAUDE.md          # 项目规范
```

## 安全

- 仅在 `127.0.0.1` 监听，不暴露到局域网
- 文件操作不经过任何云端，所有数据留在本地
- 删除操作有二次确认弹窗
- 禁止删除 Home 根目录
